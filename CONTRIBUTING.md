# Contributing to VAULTX

Thank you for your interest in contributing to VAULTX! We welcome community contributions to help improve the platform.

## 📖 Code of Conduct

By participating in this project, you agree to abide by the terms of our [Code of Conduct](CODE_OF_CONDUCT.md).

## 🚀 How to Contribute

### 1. Reporting Bugs
- Search existing issues to see if the bug has already been reported.
- If not, open a new issue detailing steps to reproduce, expected behavior, and screenshots if applicable.
- For security-related bugs, please refer strictly to our [Security Policy](SECURITY.md) and do not open a public issue.

### 2. Feature Requests
- Check if your feature idea is already requested or discussed.
- Open an issue explaining the proposed feature, the use cases, and how it aligns with VAULTX's tech philosophy.

### 3. Submitting Pull Requests
- Fork the repository and create your branch from `main`.
- Keep your changes atomic and targeted to a single issue.
- Ensure that the project compiles cleanly and passes type checking:
  ```bash
  npm run typecheck
  npm run build
  ```
- Write clear commit messages and describe the changes in detail in your Pull Request.
